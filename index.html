01
Stack technique recommandé
⚠️
Contraintes RDC : internet instable → PWA obligatoire (offline-first). Mobile Money (MTN/Airtel/Orange) au lieu de Stripe. Hébergement africain ou CDN proche pour la latence.
Frontend
Next.js 14 (App Router)
SSR + SSG + PWA. React Server Components. Le prototype DC est le guide UI exact.
Alt: Nuxt.js 3
CSS
TailwindCSS + shadcn/ui
Classes utilitaires. shadcn/ui donne les composants (Dialog, Table, Form) prêts à l'emploi, accessibles.
Alt: MUI / Mantine
Backend
Node.js + Fastify
API REST rapide. Léger en mémoire (important sur serveurs bon marché). Plugins natifs pour auth, validation.
Alt: Express.js
Base de données
PostgreSQL + Prisma ORM
Relationnelle → parfait pour les 13 modules liés. Prisma génère les types TypeScript auto.
Alt: MySQL / Supabase
Auth
JWT + bcrypt
Simple, sans dépendance externe. Le RBAC (5 rôles) se gère manuellement dans un middleware léger.
Alt: NextAuth.js
Paiements RDC
FlexPay API (Kinshasa)
MTN Money, Airtel Money, Orange Money. API locale RDC, documentation en français, sandbox disponible.
Alt: CinetPay
PDF
React-PDF / Puppeteer
Génération devis, factures, contrats. React-PDF = côté serveur léger. Puppeteer = rendu fidèle HTML.
Alt: PDFKit
Déploiement
Railway.app ou Render.com
Hébergement Node + PostgreSQL tout-en-un. CI/CD depuis GitHub. Gratuit pour commencer, pay-as-you-grow.
Alt: DigitalOcean / VPS
PWA / Offline
next-pwa + IndexedDB
Cache des listes (athlètes, clients) pour usage sans internet. Sync différée quand connexion rétablie.
Alt: Workbox

Guide Claude Code — Étape par étape
Comment passer du prototype au code de production avec Claude Code
01
Initialisation du projet
~30 min
Créez la structure Next.js + backend Fastify. Claude Code génère l'arborescence complète en un prompt.
Crée un monorepo avec : apps/web → Next.js 14 App Router + TailwindCSS + shadcn/ui apps/api → Fastify + Prisma + PostgreSQL packages/types → types partagés TypeScript Structure apps/api : src/modules/{auth,athletes,clients,devis,factures,projets,contrats,settings}
02
Schéma Prisma + seed
~45 min
Collez le schéma Prisma de ce document dans schema.prisma. Claude Code génère les migrations et un seed réaliste.
Voici le schéma Prisma (copie depuis handoff) : [coller le schéma complet] 1. npx prisma migrate dev --name init 2. Génère seed.ts avec 5 athlètes, 5 clients, 4 devis et 1 admin (données réalistes RDC — football, basketball, Kinshasa)
03
Module Auth + RBAC
~1h
JWT avec refresh tokens. Middleware de vérification de rôle. 5 rôles : ADMIN, MANAGER, COMMERCIAL, COACH, COMPTABLE.
Crée le module auth Fastify : - POST /auth/login → JWT access (15min) + refresh (7j) HttpOnly cookie - Middleware checkRole(roles: Role[]) → 403 si non autorisé - Guard sur chaque route selon la matrice RBAC du handoff - bcrypt pour les mots de passe
04
CRUD Athlètes + Clients
~2h
Les deux modules cœur métier. Utilisez le prototype comme référence visuelle exacte pour les champs.
Génère le module Athlètes complet : Backend : GET/POST/PUT/DELETE /api/athletes avec pagination, search, filter par sport Frontend : page /athletes avec : - Table (colonnes : nom, sport, poste, niveau, club, valeur_marchande) - Sheet/drawer latéral au clic → détail complet + actions - Formulaire création/édition Utilise le composant Table de shadcn/ui. Respecte la palette du design system.
05
Pipeline commercial (Devis → Facture)
~3h
Le flux le plus complexe. Devis → conversion en Facture → Règlement. Numérotation auto DEV-YYYY-NNN.
Implémente le flux Devis → Facture : 1. Création devis avec lignes de prestations (JSON : [{prestationId, quantite, prixUnit}]) 2. Numérotation auto depuis Settings (préfixe + année + séquence) 3. POST /devis/:id/convert → crée Facture liée, statut Devis = CONVERTI 4. Génération PDF avec React-PDF (logo, infos légales de Settings, lignes, total) 5. Page facture avec bouton "Enregistrer paiement" → POST /reglements
06
Projets + Kanban drag & drop
~2h
Kanban 4 colonnes. Utiliser @dnd-kit/core pour le drag & drop. PATCH /taches/:id/move pour persister la position.
Crée le Kanban Projets avec @dnd-kit/core : - 4 colonnes : TODO | EN_COURS | EN_ATTENTE | TERMINÉ - Drag & drop entre colonnes → PATCH /taches/:id/move {colonne, position} - Optimistic updates (UI immédiat, rollback si erreur API) - Filtre par projet dans le header
07
Mobile Money (FlexPay)
~2h
Intégration FlexPay pour MTN Money, Airtel Money, Orange Money. Webhook pour confirmation de paiement.
Intègre FlexPay API (flexpay.cd) : 1. POST /reglements/initiate → appel FlexPay, retourne orderNumber 2. GET /reglements/status/:orderNumber → vérifie statut 3. POST /webhooks/flexpay → reçoit confirmation, met à jour Facture.statut Variables d'env : FLEXPAY_TOKEN, FLEXPAY_MERCHANT_CODE Sandbox URL : https://backend.flexpay.cd/api/rest/v1/paymentService
08
Déploiement Railway.app
~1h
CI/CD depuis GitHub. PostgreSQL géré par Railway. Variables d'environnement sécurisées.
Configure le déploiement Railway : 1. railway.toml pour apps/api (build command + start command) 2. next.config.js pour le frontend (output: standalone) 3. Variables d'env nécessaires : DATABASE_URL, JWT_SECRET, FLEXPAY_TOKEN, NEXT_PUBLIC_API_URL 4. Dockerfile multi-stage pour l'API (node:20-alpine) 5. Script de migration auto au démarrage (prisma migrate deploy)

Risques techniques RDC
📶
Internet instable
Coupures fréquentes à Kinshasa. Les pages doivent rester utilisables offline.
→ PWA + cache Athlètes/Clients en IndexedDB. Sync au retour réseau.
📱
Mobile-first
L'équipe utilise principalement des téléphones Android sur le terrain.
→ Design responsive. Tester sur Chrome Android. Min 44px pour les zones cliquables.
⚡
Coupures de courant
Pannes d'électricité fréquentes → risque perte de données non sauvegardées.
→ Auto-save formulaires toutes les 30s dans localStorage. Indicateur de sauvegarde visible.
💰
Mobile Money instable
Les APIs FlexPay/MTN ont des timeouts. Les paiements peuvent rester en état "pending".
→ Webhook de confirmation + polling de statut toutes les 2min. UI claire pour le statut en attente.
🔐
Données sensibles
Valeurs marchandes athlètes, contrats, données financières — données critiques.
→ Chiffrement DB au repos. Logs d'audit sur actions sensibles. Accès Comptable en lecture seule.
🌍
Latence internationale
Serveurs US/EU → latence 200-400ms depuis Kinshasa sur API calls.
→ Railway région EU-West (Paris). Cloudflare CDN. Réduire le nombre de requêtes par page (React Query).
